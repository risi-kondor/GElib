GElib¶

GElib is a C++ library with a CUDA backend for performing certain operations related to the rotation group . GElib is written by Risi Kondor at the University of Chicago and Erik H Thiede at the Flatiron Institute. GElib is released under the Mozilla public license v.2.0.

This document provides documentation for GElib’s Python interface. Not all features in the C++ library are available through this interface. The documentation of the C++ API can be found in pdf format in the package’s doc directory.

Features¶

Installation¶

Installing GElib requires the following:

  1. C++11 or higher

  2. Python

  3. pybind11 (comes with PyTorch)

  4. cnine (see below)

  5. PyTorch (optional)

To install GElib follow these steps:

  1. Download the cnine and GElib libraries.

  2. Edit the file config.txt as necessary, in particular, make sure that CNINE_ROOT points to the root of the cnine package on your system.

  3. Run python setup.sty install in the python directory to compile the package and install it on your system.

To use GElib from Python, load the corresponding module the usual way with import GElib. In the following we assume that from GElib import * has been called, obviating the need to prefix all GElib classes with GElib.. For matrix/tensor functionality the cnine module must also be loaded.

Known issues¶

GPU functionality is temporarily disabled.


SO3 classes¶

The fundamental classes in GElib for storing and manipulating SO(3)-equivariant vectors are SO3vec, SO3part and SO3type.

SO3vec stores a vector that the group SO(3) acts on. Any such vector can be decomposed into a sequence of smaller vectors that acts on in an irreducible way, i.e., vectors that transform as for one of the irreducible representations of .

SO3part collects those components of an SO3-vector that transform according to the same irreducible representation. Finally, SO3type is a vector of integers that specifies how many irreducible components corresponding to are featured in a given SO3-vector.


SO3type¶

An SO3type object specifies the multiplicites of each irreducible representation .

>>> tau=SO3type([3,4,1]) # Define tau=(3,4,1)
>>> tau
<GElib::SO3type(3,4,1)>
>>> print(tau)
(3,4,1)
>>> tau[2]               # Get tau_2
1
>>> tau[2]=3             # Set tau_2=3
>>> tau
<GElib::SO3type(3,4,3)>
>>> tau.maxl()           # Print out the index of the largest irrep
2

The function CGproduct computes the type of the Clebsch–Gordan product of two SO-vectors of given types.

>>> tau=SO3type([2,2])
>>> mu=CGproduct(tau,tau)
>>> print(mu)
(8,12,4)

The optional argument maxl limits the product to a given maximal .

>>> tau=SO3type([1,1,1])
>>> print(CGproduct(tau,tau))
(3,6,6,3,1)
>>> print(CGproduct(tau,tau,2))
(3,6,6)

SO3part¶

An SO3part objects stores n different vectors transforming accoring to the same irreducible representation . Since is a dimensional representation, the vectors can be jointly stored in a matrix .

The following constructs an SO3part object holding n=3 vectors corresponding to the l=2 irrep.

>>> A=SO3part.gaussian(2,3)
>>> A
GElib::SO3part(l=2,n=3)
>>> print(A)
[ (-0.302782,0.137796) (-0.999364,0.674001) (-1.62807,-0.281158) ]
[ (-0.250657,-1.1945) (0.658802,1.06918) (-0.973458,-1.2115) ]
[ (-1.20183,-2.1947) (-0.399454,0.680457) (-0.727057,1.40633) ]
[ (0.43853,-0.31774) (-0.42954,0.123568) (-2.20967,-0.37048) ]
[ (-1.22569,0.0266631) (0.73464,0.592888) (0.630166,-0.165017) ]

The order l and multiplicity n of an SO3part is accessed as follows.

>>> A=SO3part.gaussian(2,3)
>>> A.getl()
2
>>> A.getn()
3

[ (-0.00645578,-1.14759) (1.30118,-0.668608) (-2.32167,-1.85946) ]

>>> print(A-B)

Individual entries in an SO3part can be accessed with the same syntax as how matrix elements are accessed. Note however the that indexing convention is that n index comes first, and the m index corresponding to indexing the components within a given irreducible vector is second.

It is also important to note that in general manually setting entries in SO3part objects individually breaks equivariance.

>>> A=SO3part.gaussian(2,3)
>>> A(2,3)
(1.4733880758285522+0.1949467808008194j)
>>> A[2,3]
(1.4733880758285522+0.1949467808008194j)
>>> A[2,3]=1.0
>>> print(A)
[ (-1.23974,1.16493) (-0.407472,0.584898) (1.61201,-0.660558) ]
[ (0.399771,0.534755) (1.3828,-0.607787) (0.0523187,0.74589) ]
[ (-0.904146,-1.75177) (1.87065,-0.965146) (-1.66043,-0.474282) ]
[ (-0.688081,-0.546571) (0.0757219,-0.0384917) (1,0) ]
[ (0.097221,-0.485144) (-0.89237,-0.370271) (-0.228782,-1.12408) ]

Functions of SO3-parts¶

Similarly to tensors, it is possible to take the inner products and norms of SO3part objects.

>>> from GElib import *
>>> A=SO3part.gaussian(2,3)
>>> B=SO3part.gaussian(2,3)
>>> inp(A,B)
(1.5953152179718018+5.115486145019531j)
>>> norm2(A)
7.411661148071289

Matrix operations¶

SO3part objects can also be multiplied by cnine::ctensor matrices from the right. Multiplication by matrices from the left is not allowed because it would break equivariance. Note that to construct cnine tensors, first the cnine module must be loaded.

>>> from GElib import *
>>> A=SO3part.gaussian(2,3)
>>> from cnine import *
>>> M=ctensor.gaussian([3,3])
>>> B=A*M
>>> print(B)
[ (6.03209,0.449935) (-0.717159,1.11423) (-4.45347,-1.60968) ]
[ (-1.65973,0.693803) (1.65098,-1.79472) (2.36366,-1.61901) ]
[ (-2.75144,0.117771) (1.33895,-3.43707) (9.36576,1.70359) ]
[ (2.5369,3.65761) (-0.0298907,-0.733082) (-0.894675,-1.63206) ]
[ (1.22471,-1.53125) (-1.74749,0.0933496) (-1.33813,1.35488) ]

Spherical harmonics¶

SO3part objects can be initialized as spherical harmonic coefficients of a three dimensional vector correspnding to a given .

>>> A=SO3part.spharm(2,[0.12,0.31,-0.55])
>>> print(A)
[ (-0.0764131,-0.0695855) ]
[ (-0.123458,0.318933) ]
[ (0.37763,0) ]
[ (0.123458,0.318933) ]
[ (-0.0764131,0.0695855) ]

Clebsch-Gordan products¶

GPU operations¶

Similarly to cnine tensors, SO3part objects can moved back and forth between the host (CPU) and the GPU with the to method.

>>> A=SO3part.gaussian(4,4)
>>> B=A.to(1) # Create a copy of A on the first GPU (GPU0)
>>> C=B.to(0) # Move B back to the host

SO3vec¶

An SO3vec object represents a general SO(3)-covariant vector. On the backend, SO3vec is stored as a collection of SO3part objects with .

An SO3vec can be initialized from an SO3type object or just a vector of multiplicities corresponding to .

>>> tau=SO3type([2,3,1])
>>> v=SO3vec.gaussian(tau)
>>> v=SO3vec.gaussian([2,3,1])
>>> v
<GElib::SO3vec of type(2,3,1)>
>>> print(v)
Part l=0:
  [ (-1.50279,-0.929941) (0.570759,-0.934988) ]

Part l=1:
  [ (-0.764676,0.537104) (0.250854,0.694816) (-0.188164,0.541231) ]
  [ (-1.51315,-1.96886) (1.32256,0.354178) (1.93468,-0.455262) ]
  [ (1.25244,0.377994) (1.0417,-0.274068) (-0.696964,0.005616) ]

Part l=2:
  [ (-1.77286,-1.83641) ]
  [ (0.519691,-0.257851) ]
  [ (0.0431933,-0.391737) ]
  [ (-1.96668,2.69588) ]
  [ (-0.480737,1.6585) ]

The type of an SO(3)-vector and the total number of parts can be accessed as follows.

>>> v.type()
<GElib::SO3type(2,3,1)>
>>> len(v)
3

Accessing parts and arithmetic¶

Individual parts of the vector can be accessed as follows.

>>> A=v[1]
>>> print(A)
[ (1.3828,-0.89237) (0.0523187,-0.228782) (-0.904146,1.16493) ]
[ (1.87065,0.584898) (-1.66043,-0.660558) (-0.688081,0.534755) ]
[ (0.0757219,-0.607787) (1.47339,0.74589) (0.097221,-1.75177) ]

>>> v[1]=SO3part.ones(1,2)
>>> print(v)
Part l=0:
  [ (-0.0893479,0.200541) (2.04214,0.277807) ]

Part l=1:
  [ (1,0) (1,0) ]
  [ (1,0) (1,0) ]

Functions of SO3vec objects¶

Arithmetic operations can be applied to SO3vec objects just as to SO3part s.

>>> u=SO3vec.gaussian([1,1])
>>> v=SO3vec.gaussian([1,1])
>>> inp(u,v)
(5.523734092712402-1.4036915302276611j)
>>> norm2(u)
(10.693071365356445+0j)

Clebsch-Gordan products¶

The Clebsch-Gordan product of two SO3-vectors can be computed as follows.

>>> u=SO3vec.gaussian([1,1])
>>> v=SO3vec.gaussian([1,1])
>>> w=CGproduct(u,v)
>>> print(w)
Part l=0:
  [ (0.800454,-2.72231) (0.387997,-2.21325) ]

Part l=1:
  [ (-1.08378,0.166964) (-1.13947,1.02458) (-0.979756,-0.170846) ]
  [ (-3.14667,-0.020229) (-1.60544,-0.595765) (-0.658927,1.13758) ]
  [ (0.573493,3.50629) (0.609701,-0.290724) (-1.86063,-0.256204) ]

Part l=2:
  [ (0.545523,0.23039) ]
  [ (1.0578,1.10345) ]
  [ (0.098245,0.754121) ]

GPU operations¶

Similarly to cnine tensors, SO3part objects, SO3vec objects

can moved back and forth between the host (CPU) and the GPU with the to method.

>>> A=SO3vec.gaussian([2,3,1])
>>> B=A.to(1) # Create a copy of A on the first GPU (GPU0)
>>> C=B.to(0) # Move B back to the host
