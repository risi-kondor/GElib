GElib¶

GElib is a C++ library with a CUDA backend for computations related to the rotation group SO(3), specifically for building SO(3)-equivariant neural networks. GElib is developed by Risi Kondor at the University of Chicago. GElib is released under the Mozilla public license v.2.0.

This document provides documentation for GElib’s Python interface. Not all features in the C++ library are available through this interface. The documentation of the C++ API can be found in pdf format in the package’s doc directory.

Features¶

  1. Classes to store and manipulate SO(3)-equivariant vectors

  2. Fast implementation of the Clebsch-Gordan transform on both the CPU and the GPU

  3. Facilities for operating on arrays of SO(3)-equivariant vectors in parallel, even in irregular patterns (graphs)

  4. Support for automatic differentiation

  5. Interoperability with PyTorch

Installation¶

Installing GElib requires the following:

  1. C++14 or higher

  2. Python

  3. PyTorch

  4. cnine (see below)

To install GElib follow these steps:

  1. Download the cnine and GElib libraries.

  2. Edit the file config.txt as necessary, in particular, make sure that CNINE_ROOT points to the root of the cnine package on your system.

  3. Run python setup.sty install in the python directory to compile the package and install it on your system.

Usage¶

GElib has two distinct interfaces implemented in two different modules:

  1. To use the library without PyTorch’s autodiff functionality, load the library with import gelib_base as gelib.

  2. To use the library with automatic differentiation, load the library with import gelib_torch as gelib.

The two modules use identical syntax, therefore the following description of their usage applies to both. The backend implementation of the two modules however is quite different. gelib_base is just a wrapper for the underlying C++ classes. In contrast, gelib_torch ‘s core classes are Python classes derived from torch.tensor for interoperability with torch.autodiff. These Python classes then, in turn, call the wrappers implemented in gelib_base. Inevitably, the latter approach incurs some performance overhead. Since gelib_torch is built on gelib_base, so the two modules can also be used together.

Known issues¶

GPU functionality is temporarily disabled.


SO3 classes¶

The core classes in GElib for storing and manipulating SO(3)-equivariant vectors are SO3part and SO3vec.

SO3part stores vectors that transform according to a specific irreducible representation of SO(3).

In contrast SO3vec stores a vector that transforms according to a generic representation. Since any representation decomposes into a sum of irreducibles, the vector is actually stored as a sequence of SO3part objects.

The library also provides the SO3element and SO3type helper classes.


SO3element¶

An SO3element object represents a single group element . The group element can either be initialized from the Euler angles or chosen randomly from the group.

>>> a=SO3element(0.2,1.0,-0.1)
>>> a
SO3element(0.200000,1.000000,-0.100000)
>>> b=SO3element.uniform()
>>> b
SO3element(0.534275,2.801079,1.191856)

SO3type¶

An SO3type object specifies the multiplicites of each irreducible representation in an SO3-vector.

>>> tau=SO3type([3,4,1]) # Define tau=(3,4,1)
>>> tau
<GElib::SO3type(3,4,1)>
>>> print(tau)
(3,4,1)
>>> tau[2]               # Get tau_2
1
>>> tau[2]=3             # Set tau_2=3
>>> tau
<GElib::SO3type(3,4,3)>
>>> tau.maxl()           # Print out the index of the largest irrep
2

The function CGproduct computes the type of the Clebsch–Gordan product of two SO-vectors of given types.

>>> tau=SO3type([2,2])
>>> mu=CGproduct(tau,tau)
>>> print(mu)
(8,12,4)

The optional argument maxl limits the product to a given maximal .

>>> tau=SO3type([1,1,1])
>>> print(CGproduct(tau,tau))
(3,6,6,3,1)
>>> print(CGproduct(tau,tau,2))
(3,6,6)

SO3part¶

An SO3part objects stores n different vectors transforming accoring to the same irreducible representation . Since is a dimensional representation, the vectors are jointly stored in a matrix .

The following constructs an SO3part object holding n=3 vectors corresponding to the l=2 irrep.

>>> A=SO3part.gaussian(2,3)
>>> A
GElib::SO3part(l=2,n=3)
>>> print(A)
[ (-0.302782,0.137796) (-0.999364,0.674001) (-1.62807,-0.281158) ]
[ (-0.250657,-1.1945) (0.658802,1.06918) (-0.973458,-1.2115) ]
[ (-1.20183,-2.1947) (-0.399454,0.680457) (-0.727057,1.40633) ]
[ (0.43853,-0.31774) (-0.42954,0.123568) (-2.20967,-0.37048) ]
[ (-1.22569,0.0266631) (0.73464,0.592888) (0.630166,-0.165017) ]

The order l and multiplicity n of an SO3part is accessed as follows.

>>> A=SO3part.gaussian(2,3)
>>> A.getl()

>>> A.getn()

Individual entries in an SO3part can be accessed with the same syntax as how matrix elements are accessed in cnine. Note however the that indexing convention is that n index comes first, and the m index corresponding to the index within a given irreducible vector is second.

>>> A=SO3part.gaussian(2,3)
>>> A(2,3)
(1.4733880758285522+0.1949467808008194j)
>>> A[2,3]
(1.4733880758285522+0.1949467808008194j)
>>> A[2,3]=1.0
>>> print(A)
[ (-1.23974,1.16493) (-0.407472,0.584898) (1.61201,-0.660558) ]

Functions of SO3-parts¶

Similarly to tensors, it is possible to take the inner products and norms of SO3part objects.

>>> from GElib import *
>>> A=SO3part.gaussian(2,3)
>>> B=SO3part.gaussian(2,3)
>>> inp(A,B)
(1.5953152179718018+5.115486145019531j)
>>> norm2(A)
7.411661148071289

Matrix operations¶

SO3part objects can be multiplied by cnine::ctensor matrices from the right. Multiplication by matrices from the left is not allowed because it would break equivariance. Note that to construct cnine tensors, first the cnine module must be loaded.

>>> from GElib import *
>>> A=SO3part.gaussian(2,3)
>>> from cnine import *
>>> M=ctensor.gaussian([3,3])
>>> B=A*M
>>> print(B)
[ (6.03209,0.449935) (-0.717159,1.11423) (-4.45347,-1.60968) ]
[ (-1.65973,0.693803) (1.65098,-1.79472) (2.36366,-1.61901) ]

GPU operations¶

Similarly to cnine tensors, SO3part objects can moved back and forth between the host (CPU) and the GPU with the to method.

>>> A=SO3part.gaussian(4,4)
>>> B=A.to(1) # Create a copy of A on the first GPU (GPU0)
>>> C=B.to(0) # Move B back to the host

SO3vecArr¶

An SO3vecArr is a multidimensional array of SO3vec objects. On the backend, the SO3vecArr is stored as a list of GElib::SO3partArray objects, however logically it is accessible as a single array of SO3vec objects.
